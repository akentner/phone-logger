1. `test_app` fixture (async):
   - Creates FastAPI app via `create_app(lifespan=None)` (no lifespan context — we don't want real MQTT/Fritz startup)
   - Registers routes: resolve, contacts, calls, pbx, cache, config_routes, i18n, gui_router (from src.api.routes and src.gui.routes)
   - Injects `test_db` fixture via FastAPI dependency overrides
   - Injects mocked `get_pipeline()` with a fake PbxStateManager that has `_devices_by_id = {}`
   - Returns the app ready for TestClient

2. `test_client` fixture (async):
   - Uses `AsyncClient` from httpx with `ASGITransport(app=test_app)` (does NOT use `TestClient` — we need async/await for async route handlers)
   - Returns the client ready for test methods to call `await client.get()`, `await client.post()`, etc.

**Helper function** `_make_pbx_mock()`:
   - Returns a MagicMock that implements `pbx._devices_by_id = {}`, `pbx.get_status()` returning skeleton structure with empty lines/trunks/msns/devices
   - Implement `get_status()` to return: `{"lines": [], "trunks": [], "msns": [], "devices": []}`
   - This avoids PBX initialization overhead while allowing routes that reference `pbx._devices_by_id` to not crash

**Use FastAPI dependency override pattern** (from FastAPI docs):
```python
app.dependency_overrides[get_db] = lambda: test_db_instance
app.dependency_overrides[get_pipeline] = lambda: mock_pipeline_with_pbx
```

Do NOT modify the lifespan context manager in src/main.py — just bypass it by passing `lifespan=None` to `create_app()`.

Note: `create_app()` is imported from `src.api.app`, not `src.main.create_application()`.
