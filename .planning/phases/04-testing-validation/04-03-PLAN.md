**Test: test_connect_without_prior_disconnect_completes_normally**
- Scenario: CONNECT arrives, then another CONNECT for the same connection_id (no DISCONNECT between)
- Setup: Create CallLogOutputAdapter with test_db
- Step 1: RING event with connection_id="100" → creates call record
- Step 2: CONNECT event with connection_id="100" → updates call.connected_at
- Step 3: CONNECT event again with connection_id="100" → second time
- Step 4: DISCONNECT with connection_id="100" → ends call
- Verify:
  - All 4 events process without exception
  - Final call has status "answered"
  - Duration calculated correctly from first connected_at
  - No crash when CONNECT is duplicated

**Test: test_missing_ring_missing_connect_ends_in_unknown_state**
- Scenario: CALL event (outbound), then immediately DISCONNECT without CONNECT
- Setup: Create CallLogOutputAdapter
- Step 1: CALL event with connection_id="101"
- Step 2: DISCONNECT with connection_id="101"
- Verify:
  - Call created with status "ringing" (from CALL event)
  - DISCONNECT updates status to "notReached" (no CONNECT → not reached)
  - duration_seconds == 0 (no answer period)
  - No exception
