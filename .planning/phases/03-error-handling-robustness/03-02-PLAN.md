Add field count validation to the Fritz!Box Callmonitor line parser. The current parser does a single global `len(parts) < 4` check and then accesses fields by index with optional guards. Purpose: Malformed Fritz!Box messages (firmware updates, unusual network events, truncated TCP reads) will now be caught early, logged with the raw line for debugging, and rejected before field access. Makes parse failures observable.

Output: `MIN_FIELDS` constant added to fritz_callmonitor.py, single validation guard inserted in `_parse_line()` after event_type resolution. New tests covering all field count failure paths.

```python
# Minimum required field count per event type (including date field at index 0).
# RING:       date;RING;conn;caller;called             → 5
# CALL:       date;CALL;conn;ext;caller;called         → 6
# CONNECT:    date;CONNECT;conn;ext;number             → 5
# DISCONNECT: date;DISCONNECT;conn;duration            → 4
MIN_FIELDS: dict[CallEventType, int] = {
    CallEventType.RING: 5,
    CallEventType.CALL: 6,
    CallEventType.CONNECT: 5,
    CallEventType.DISCONNECT: 4,
}
```

**Step B: Update _parse_line() — two changes**

Change 1: In the `if not event_type:` block, change silent `return None` to log at DEBUG first:
```python
        event_type = EVENT_MAP.get(event_name)

            return None
```

Change 2: Insert the MIN_FIELDS validation immediately AFTER the `if not event_type:` block and BEFORE the timestamp parsing `try:` block. Do NOT modify any other part of the method:

```python
        if len(parts) < MIN_FIELDS[event_type]:
            logger.warning(
                "Fritz parse error [%s]: need %d fields, got %d | raw: %s",
                event_type.value,
                MIN_FIELDS[event_type],
                len(parts),
                line,
            )
            return None
```

**The existing `if len(parts) < 4: return None` pre-check at the top of _parse_line() stays unchanged.** It acts as a guard before parts[1] is accessed. The MIN_FIELDS check is additive, not a replacement.

**All existing `if len(parts) > N` guards in the per-event-type blocks stay unchanged** — they handle optional trailing fields (SIP line in RING/CALL). The MIN_FIELDS check only validates the mandatory minimum.
