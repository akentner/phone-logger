Run the following command to add ruff and pytest-cov as dev dependencies:

```bash
uv add --dev "ruff>=0.15.10" "pytest-cov>=7.1.0"
```

NOTE: Do NOT add a separate `ruff-format` package. The `ruff format` CLI subcommand ships with the `ruff` package itself. The PyPI `ruff-format` package is a PyO3 binding for programmatic use, not the CLI formatter. Adding `ruff` to dev deps satisfies D-04 for both linting and formatting.

Then edit pyproject.toml to add the following sections (append after the existing `[tool.pytest.ini_options]` block):

```toml
[tool.pytest.ini_options]
asyncio_mode = "auto"
testpaths = ["tests"]
addopts = "--cov=src --cov-report=term-missing"

[tool.ruff]
line-length = 120

[tool.ruff.lint]
select = ["E", "F", "W"]

[tool.coverage.run]
omit = ["tests/*"]
```

IMPORTANT: The `addopts` line MUST be added in the same commit as pytest-cov is installed. pytest-cov must be present in the venv before `addopts` is set, or every `uv run pytest` call will fail with "unrecognized arguments: --cov=src". The `uv add --dev pytest-cov` command above handles this.

After editing pyproject.toml, run `uv sync` to ensure the venv is up to date.

Verify the test suite still passes:
```bash
uv run pytest tests/ -q --no-header
```
All 201 tests must remain green. If any fail, the pyproject.toml edit introduced a problem — check addopts syntax.
